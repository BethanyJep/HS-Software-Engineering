# Program Overview – Alliance Girls Software Engineering Program

## About the Program

This program is a collaboration between **TOFA** and **Alliance Girls Alumni**, designed to introduce Grade 10 students at Alliance Girls High School to the world of software engineering through **project-based learning**.

The program runs across two school terms:

- **Term 2** – Foundation & Inspiration (2–3 facilitated sessions)
- **Term 3** – Project Sprint & Showcase

At the end of Term 3, student teams present their solutions and a **prize is awarded to the winning team**.

This program is also intended to serve as a **reusable roadmap** that can be adapted and rolled out at other schools.

---

## Program Timeline

### Term 2 – Foundation Sessions

| Session | Topic | Duration | Key Outcomes |
|---|---|---|---|
| 1 | Introduction to Software Engineering | 90 min | Understand what SE is; see real demos; get excited |
| 2 | Career Conversations | 90 min | Hear from alumni; explore roles in tech; visualise pathways |
| 3 | Resource Session | 90 min | Know where to learn; set up tools; form project groups |

### Term 3 – Project Sprint

| Phase | Timing | Activity |
|---|---|---|
| Kickoff | Week 1 | Teams finalized; problem statements submitted |
| Ideation | Weeks 1–2 | Research, brainstorm, define solution |
| Build | Weeks 3–8 | Design UI, write code, test, iterate |
| Polish | Weeks 9–10 | Refine MVP, prepare presentation |
| Showcase | End of Term | Teams present to panel of judges |
| Prize Ceremony | Showcase Day | Winning team announced and awarded |

---

## Program Goals

1. **Inspire** – Show students that software engineering is an exciting, accessible career.
2. **Educate** – Give students a foundational understanding of how software is built.
3. **Empower** – Equip students with starter tools and resources to learn independently.
4. **Create** – Guide students through building a real solution to a real problem.
5. **Celebrate** – Recognise effort, creativity, and teamwork at the showcase.

---

## Structure & Facilitation

### Who Facilitates?
- TOFA members and Alliance Girls Alumni volunteers
- Guest speakers for Career Conversations session (can be remote/hybrid)
- Judges panel for the Term 3 showcase (alumni, industry contacts)

### Suggested Team Size
- 3–5 students per team for the Term 3 project

### Tools & Platforms
| Purpose | Suggested Tool | Notes |
|---|---|---|
| Coding | Scratch, Thunkable, or CodePen (HTML/CSS/JavaScript) | Match to skill level |
| Design | Figma (free for students) | Good for prototyping |
| Collaboration | Google Docs / Slides | Already available at school |
| Version control (advanced) | GitHub | Introduce in Session 3 |
| Communication | WhatsApp group or Google Classroom | Coordinate between sessions |

---

## Assessment / Judging Criteria (Term 3)

| Criterion | Weight | Description |
|---|---|---|
| Problem relevance | 25% | Does the solution address a real, meaningful problem? |
| Creativity & Innovation | 25% | Is the approach original and thoughtful? |
| Technical implementation | 25% | Does the software work? How well is it built? |
| Presentation | 25% | Is the pitch clear, confident, and compelling? |

## Adapting This Program for Other Schools

When rolling this out elsewhere, consider:

1. **Assess existing resources** – Does the school have computers? Internet access? A coding club?
2. **Adjust the tools** – Use offline-friendly tools (e.g., Scratch offline) if connectivity is limited.
3. **Scale the sessions** – More sessions may be needed if students have less prior exposure to tech.
4. **Recruit local alumni** – Career Conversations land better when the speaker went to the same school.
5. **Partner with a sponsor** – A prize makes the showcase event feel significant and motivates effort.
